Nginx Location Block

Syntax

location [=|^~|~|~*|@] { ... }

There are 6 possible modifiers for the location block (including the no-moddifer option).

Modifier Description
none Exact match or begins with.
= Exact match.
^~ Begins with (higher precedence than no modifier).
~ Regex.
~* Case-insensitive regex.
@ Named location block.

Matching precedence

  1. = try exact match.
  2. none try exact match without modifier.
  3. ^~ try begins with.
  4. ~ try regex.
  5. ~* try case-insensitive regex.
  6. none try begins with.

The order of location blocks doesn't not matter, even when ^~ is present.
